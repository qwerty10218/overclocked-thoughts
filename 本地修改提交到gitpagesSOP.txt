第一步：改代碼 (Coding)
改到你爽為止。

第二步：終端機 npm run deploy (先出貨)
為什麼？因為你想趕快看到網站變更。

執行這個，GitHub 網站就會更新。

如果這裡報錯（例如語法錯誤），你馬上就知道，不用急著存檔。

第三步：左邊介面 提交 & 同步 (後存檔)
確認網站沒問題了，再把這份完美的代碼備份到雲端。

🔥 Gary 的最終極簡 SOP (只要記這個就好)
以後不管你要修復白屏、改文字、換圖片，完全不要思考，照著這 3 個動作 做就對了：

去改代碼 (比如現在去 vite.config.js 刪掉那個尖括號)。

開終端機輸入：
npm run deploy
(等它跑完出現 Published)

去左邊介面按： 提交 -> 同步變更

就這樣！現在去執行第 1 步和第 2 步，把那個尖括號修好，你的網站就復活了！
